# CPCS-MX Schema Field Guide

## The four primary artifacts

| Artifact | Purpose | Authoritative for |
|---|---|---|
| Authoring YAML | readable profiles, direction, overrides | human/agent intent before resolution |
| Canonical JSON | resolved score | semantics, IDs, clocks, constraints, compilation |
| JSONL | observations, RAG, metrics, audit | independent evidence and events |
| Dense assets | sampled geometry and coefficients | frame/sample-level numerical control |

XML may wrap an agent request or semantic envelope. It is not used for dense motion.

## Top-level canonical fields

- `schema_id`: exact CPCS-MX schema version.
- `document_id`: stable score ID.
- `timebase`: canonical clock, FPS, sample rate, interval semantics.
- `coordinate_systems`: every geometric basis used by tracks.
- `assets`: external arrays, clips, references, and caches.
- `characters`: narrative characters and profile references.
- `rigs`: skeletons, joint sets, limits, and retarget maps.
- `shots`: presentation hierarchy and beats.
- `phases`: global or local motion phases.
- `tracks`: root, joint, camera, face, breath, or derived curves.
- `events`: point or interval events.
- `contacts`: support, grasp, near-contact, or virtual collision.
- `laban_controls`: Body–Effort–Shape–Space descriptors.
- `facial_events`: FACS-like AU, gaze, and head events.
- `breath_tracks`: respiratory phrase and effort synchronization.
- `mannerism_profiles`: character-specific movement signatures.
- `secondary_motion`: cloth, hair, accessories, and soft motion.
- `style_transforms`: typed natural/UGC/animation/anime/superhuman transformations.
- `constraints`: hard, soft, and advisory requirements.
- `verification_plan`: metrics and acceptance gates.
- `provenance`: derivation, tools, revisions, and defaults.
- `extensions`: namespaced project fields.

## Required semantic distinctions

### Root versus joints

Root tracks move and orient the character globally. Joint tracks articulate the skeleton locally. A controller-driven root and a clip-driven root require different contact policies.

### Contact versus impact presentation

A `staged_near_contact` can be accompanied by an impact flash, shake, sound, and reaction without a physical collision. Store the visual event, audio event, reaction, and spatial relation separately.

### Laban versus mechanics

Laban Effort describes perceived movement quality. Mass, force, torque, speed, and acceleration are separate. A proxy may relate them for a project but must be versioned.

### FACS versus emotion

FACS AUs code visible facial action. `displayed_affect`, `character_state`, and any interpretation of emotion are separate fields.

### Anatomical motion versus stylized deformation

Do not expand a human joint range to create cartoon stretch. Use `virtual_stylized` limits on a nonhuman rig or a mesh/perspective/smear deformation layer.

### Motion versus presentation

Camera, editing, motion blur, speed lines, holds, and smear frames can alter perceived motion without changing the staged-world trajectory.

## Authority values

- `locked`: must be satisfied within tolerance.
- `guided`: strong target; solver may deviate and report residual.
- `preferred`: weighted style or quality target.
- `free`: generated by the motion prior or solver.

## Evidence values

- `measured`
- `detected`
- `inferred`
- `interpreted`
- `authored`
- `defaulted`
- `derived`

## Constraint priorities

- `hard`: rejection or explicit authorized relaxation.
- `soft`: optimized and measured.
- `advisory`: prompt or review guidance.

## Recommended minimum canonical score

```json
{
  "schema_id": "urn:cpcs-mx:schema:1.0",
  "document_id": "score.example.001",
  "timebase": {
    "canonical": "seconds",
    "fps": {"numerator": 24, "denominator": 1},
    "sample_rate_hz": 120,
    "interval_semantics": "half_open"
  },
  "shots": [],
  "tracks": [],
  "events": [],
  "constraints": [],
  "verification_plan": {"metrics": [], "acceptance_gates": []}
}
```

A minimal score can validate structurally while still being too weak for exact generation. Capability and coverage reports remain necessary.
