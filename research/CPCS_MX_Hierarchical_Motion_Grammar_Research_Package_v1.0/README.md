# CPCS-MX Hierarchical Motion Grammar Research Package v1.0

## Purpose

This package is a **standalone research branch** of the broader Cinematic Performance Control Score work. It concentrates on exact, expressive, stylized, and virtual-superhuman character movement for animation, AI video generation, motion synthesis, and agent-driven production systems.

The main research artifact is:

```text
paper/CPCS_MX_Hierarchical_Motion_Grammar_Research_Paper.md
```

The paper proposes **CPCS-MX**, a layered motion representation that joins:

- source time, frame time, sample time, and phase;
- root translation and orientation;
- skeletal joint tracks and declared rotation conventions;
- support, contact, staged near-contact, grasp, and interaction events;
- velocity, acceleration, momentum, force, torque, and impulse estimates;
- inverse kinematics, retargeting, morphology, and joint-limit policies;
- Laban Body–Effort–Shape–Space descriptors;
- FACS-like facial action, gaze, head, blink, and breath timing;
- mannerisms, postural tone, cultural context, and movement signatures;
- natural movement, realistic UGC, feature animation, anime/sakuga, and virtual-superhuman transformations;
- cloth, hair, accessories, and other secondary motion;
- camera, editing, VFX, silhouette, and perceptual constraints;
- text-to-structured authoring, deterministic compilation, target adaptation, and closed-loop verification.

CPCS-MX is a **proposed research and engineering synthesis**, not an international standard, medical model, combat-training system, or guarantee that a generative video model will obey every field.

## Core architecture

```text
natural-language direction + profiles + measured assets
                         ↓
                 authoring YAML
                         ↓
       deterministic resolution and validation
                         ↓
                canonical CPCS-MX JSON
                         ↓
       target capability and adapter compilation
                         ↓
text / pose / rig / trajectory / mask / camera / VFX controls
                         ↓
              generated or animated result
                         ↓
       observation JSONL + compliance verification
```

The formats have separate responsibilities:

| Artifact | Responsibility |
|---|---|
| Markdown | Research, definitions, design rationale, and agent-readable guidance |
| YAML | Human- and agent-authored profiles, inheritance, substitutions, and overrides |
| JSON | Fully resolved canonical score and schema-governed contracts |
| JSONL | RAG records, observations, metrics, provenance, conflicts, and audit events |
| XML | Optional semantic request envelope and role-separated agent instruction |
| Dense arrays/video | Frame- or sample-level joint, pose, camera, mask, depth, and flow control |
| BVH/FBX | Import/export interchange rather than the complete semantic source of truth |

## Recommended agent ingestion

The simplest path is to ingest:

```text
rag/CPCS_MX_RAG_Corpus.jsonl
```

Validate each line against:

```text
schemas/CPCS_MX_RAG_Record_Schema.json
```

Recommended metadata filters include:

```text
record_type
document_id
evidence_labels
concepts
source_ids
schema_fields
safety_scope
```

Do not flatten or discard `evidence_labels`. The corpus deliberately separates established external research from proposed CPCS-MX conventions and operational examples.

## Quick validation

Requirements:

```text
Python 3.10+
PyYAML
jsonschema
NumPy for inspecting the sample dense track
```

Run the complete package validator from the package root:

```bash
python scripts/validate_cpcs_mx_package.py
```

Rebuild the RAG corpus:

```bash
python scripts/build_cpcs_mx_rag.py
```

Stream-validate the RAG corpus without loading it all into memory:

```bash
python scripts/validate_jsonl_stream.py \
  rag/CPCS_MX_RAG_Corpus.jsonl \
  --schema schemas/CPCS_MX_RAG_Record_Schema.json \
  --id-field record_id \
  --hash-field sha256 \
  --hash-content-field content \
  --duplicates error
```

Compile an authoring example:

```bash
python scripts/compile_authoring_yaml.py \
  examples/natural_walk.yaml \
  examples/compiled/natural_walk.compiled.json \
  --report examples/compiled/natural_walk.compiled.report.json \
  --allow-unresolved
```

Merge observation streams:

```bash
python scripts/merge_cpcs_mx_observations.py \
  examples/observations/motion_observations.jsonl \
  --schema schemas/CPCS_MX_Observation_Record_Schema.json \
  --output examples/observations/merged_active.jsonl \
  --conflicts examples/observations/conflicts.json \
  --all-records examples/observations/normalized_all.jsonl
```

## What the reference compiler does

`scripts/compile_authoring_yaml.py` demonstrates:

- safe YAML loading;
- restricted `profile://` resolution;
- deterministic typed merging;
- profile inheritance and local overrides;
- hash checking for imported assets;
- unit and timebase normalization;
- canonical JSON validation;
- explicit reporting of unresolved semantic work.

It intentionally does **not** claim to synthesize complete skeletal movement. A production compiler still needs motion retrieval, IK, dynamics, retargeting, or a learned motion generator.

## Worked examples

| Example | Main purpose |
|---|---|
| `natural_walk.yaml` | Support-aware natural locomotion with breath and mannerism |
| `realistic_ugc_gesture.yaml` | Imperfect, camera-aware creator performance without random instability |
| `staged_combat_exchange.yaml` | Safety-scoped near-contact action, reaction, and recovery |
| `anime_superhuman_action.yaml` | Anime timing and virtual-superhuman transformation with invariants |
| `cross_style_transform.json` | Typed transformation from one motion presentation domain to another |
| `canonical_cpcs_mx_score.json` | Resolved canonical score with external dense-track hashing |

All examples are fictional. The NumPy track is synthetic test data, not motion capture or a normative biomechanical template.

## Research and source policy

The package contains bibliographic metadata, research synthesis, schemas, examples, and scripts. It does not bundle copies of books, proprietary FACS manuals, paywalled papers, vendor SDKs, third-party video, or copyrighted character assets. Consult `references/CPCS_MX_Reference_Index.*` for source attribution and current links.

## Safety and rights scope

The staged-action material is for virtual animation, supervised screen performance analysis, and fictional production planning. It should not be used to optimize harmful real-world violence. Source-video transfer requires appropriate rights, consent, privacy handling, and identity-replacement policies.
