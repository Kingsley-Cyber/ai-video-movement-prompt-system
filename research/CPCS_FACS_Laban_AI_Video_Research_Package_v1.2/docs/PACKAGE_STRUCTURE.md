## Artifact roles

### Authoritative research

`paper/CPCS_FACS_Laban_AI_Video_Directorial_Control_Research_Paper.md` is the authoritative conceptual source. Section 30 introduces the inverse pipeline; Appendix H maps that pipeline to package artifacts.

### Standalone implementation guide

`docs/VIDEO_TO_CPCS_EXTRACTION_GUIDE.md` isolates the video-to-CPCS workflow for implementation teams. It covers temporal segmentation, evidence classes, provider orchestration, motion/contact reconstruction, camera/edit extraction, fight and UGC workflows, fusion, reverse compilation, and round-trip validation.

### RAG

`rag/CPCS_RAG_Corpus.jsonl` contains one document record, semantic paper chunks, and source records. The RAG schema is generated by `scripts/build_cpcs_rag.py` and stored under `schemas/`.

### Observation schemas

- `CPCS_Video_Observation_Record_Schema.json`: normalized line-oriented observations from local or hosted extractors.
- `CPCS_Video_Observation_Graph_Schema.json`: canonical evidence graph with segments, entities, observations, resolved claims, contradictions, CPCS projection, and validation.
- `CPCS_Video_Extraction_Record_Schema.json`: extended all-in-one extraction fixture schema retained for comprehensive project records.

### Scripts

- `build_cpcs_rag.py`: rebuilds and validates RAG artifacts.
- `build_reference_indexes.py`: rebuilds Markdown/JSON/CSV/TSV bibliography indexes.
- `extract_video_manifest.py`: hashes and probes media, creates analysis derivatives, and proposes frame-accurate scene-change candidates.
- `merge_video_observations.py`: fuses normalized observations while preserving source and conflict information.
- `validate_video_observation_graph.py`: performs structural and cross-field semantic validation.
- `video_to_cpcs_reference_pipeline.py`: provides an all-in-one comprehensive-record workflow with `probe`, `prepare`, `init-record`, and `validate` commands.

### Prompt and configuration templates

The prompt assets demonstrate XML, JSON, and YAML composition for provider-specific semantic passes. They must be adapted to the current provider contract. The pipeline YAML defines sampling tiers, extractor roles, evidence requirements, transfer policy, and compile targets.

### Examples and tests

The examples are synthetic and do not describe a real person or copyrighted scene. They demonstrate fight-scene extraction, UGC/marketing extraction, multi-provider evidence fusion, and a valid canonical graph. The smoke-test report records a local FFmpeg/FFprobe preprocessing and merge/validation test.
